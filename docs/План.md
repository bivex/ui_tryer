# План улучшения алгоритмов Pixel Perfect Inspector

## 🎯 Цель
Сделать ElementAnalyzer безупречным инструментом для обнаружения UI/UX проблем, основываясь на современных стандартах дизайна, Tailwind CSS и WCAG.

---

## 📊 Текущее состояние анализатора

### Что проверяется сейчас:
1. **Spacing** - соответствие отступов сетке дизайна (4px grid)
2. **Sizing** - минимальные размеры кликабельных элементов (44px)
3. **Typography** - минимальный размер шрифта (14px для мобильных)
4. **Colors** - соответствие палитре Tailwind CSS
5. **Accessibility** - базовая контрастность (4.5:1)

### Проблемы текущих алгоритмов:
- ❌ Много false positives (ложных срабатываний)
- ❌ Пропускает сложные проблемы (визуальная иерархия, выравнивание)
- ❌ Не учитывает контекст элемента и его окружение
- ❌ Нет проверки состояний (hover, focus, active)
- ❌ Нет анализа взаимосвязей между элементами

## 🚀 Улучшенные алгоритмы

## 1. SPACING ANALYSIS 2.0 (Умный анализ отступов)

### 1.1 Оптическое выравнивание (Optical Alignment)
**Проблема:** Математически центрированные элементы могут выглядеть смещенными из-за визуального веса.

**Алгоритм:**
```
1. Определить визуальный центр элемента (не геометрический):
   - Для текста: учитывать descenders (хвосты букв g, y, p)
   - Для иконок: учитывать пустое пространство в SVG
   - Для изображений: учитывать визуальный вес (темные области тяжелее)

2. Вычислить оптический сдвиг:
   opticalShift = geometricCenter - visualCenter

3. Проверить корректность выравнивания:
   if (actualOffset != opticalShift ± tolerance) {
     Issue: "Элемент математически центрирован, но оптически смещен на {delta}px"
     Suggestion: "Добавьте padding-top: {opticalShift}px для оптического центрирования"
   }
```

**Примеры применения:**
- Кнопки с иконками (иконка визуально тяжелее текста)
- Круглые аватары (темные фото визуально тяжелее светлых)
- Выпадающие списки (треугольник стрелки должен быть оптически центрирован)

---

### 1.2 Вертикальный ритм (Vertical Rhythm)
**Проблема:** Элементы расположены хаотично по вертикали, нарушая визуальный ритм.

**Алгоритм:**
```
1. Собрать все вертикальные отступы на странице:
   verticalSpacings = [margin-top, margin-bottom, padding-top, padding-bottom]

2. Определить базовую линию (baseline):
   baseline = gcd(verticalSpacings) // Наибольший общий делитель
   OR
   baseline = mostCommonValue(verticalSpacings)

3. Проверить соответствие ритму:
   for each spacing in verticalSpacings:
     if (spacing % baseline != 0) {
       Issue: "Отступ {spacing}px нарушает вертикальный ритм (baseline: {baseline}px)"
       Suggestion: "Используйте {round(spacing / baseline) * baseline}px"
     }

4. Проверить прогрессию отступов (1x, 2x, 4x, 8x baseline):
   expectedRatios = [1, 2, 4, 8, 16]
   if (spacing / baseline NOT IN expectedRatios) {
     Warning: "Отступ {spacing}px не следует прогрессии (1x, 2x, 4x, 8x)"
   }
```

**Дополнительные проверки:**
- Элементы одного уровня должны иметь одинаковые отступы
- Родственные элементы должны использовать кратные отступы
- Заголовки должны иметь больше отступа сверху, чем снизу (ratio 1.5:1 или 2:1)

---

### 1.3 Интеллектуальная проверка margin collapse
**Проблема:** Margin collapse создает неожиданные отступы.

**Алгоритм:**
```
1. Определить случаи margin collapse:
   - Вертикальные margins соседних элементов
   - Родитель + первый/последний ребенок
   - Пустые блоки с margin

2. Вычислить эффективный отступ:
   effectiveMargin = max(margin1, margin2) // не сумма!

3. Сравнить с ожидаемым:
   expectedMargin = margin1 + margin2
   if (effectiveMargin < expectedMargin) {
     Warning: "Margin collapse: ожидается {expectedMargin}px, но из-за collapse будет {effectiveMargin}px"
     Suggestion: "Используйте padding или gap вместо margin для предсказуемости"
   }

4. Проверить на нулевые/отрицательные margins:
   if (margin < 0) {
     Error: "Отрицательный margin может вызвать наложение элементов"
   }
```

---

### 1.4 Проверка избыточных отступов (Spacing Redundancy)
**Алгоритм:**
```
1. Обнаружить дублирование:
   if (element.margin == parent.padding) {
     Warning: "Дублирование отступа: margin элемента и padding родителя создают двойной отступ"
   }

2. Обнаружить нулевые отступы в Flexbox/Grid:
   if (parentLayout == 'flex' || parentLayout == 'grid') {
     if (element.margin != 0 && parent.gap != 0) {
       Warning: "Используйте gap вместо margin в {parentLayout} layout"
     }
   }

3. Проверить симметрию:
   if (paddingTop != paddingBottom && element.role != 'asymmetric') {
     Info: "Асимметричный padding может быть непреднамеренным"
   }
```

## 2. SIZING ANALYSIS 2.0 (Умный анализ размеров)

### 2.1 Контекстно-зависимые размеры кликабельных элементов

**Алгоритм:**
```
1. Определить контекст элемента:
   context = analyzeContext(element):
     - desktop: minSize = 36px (мышь точнее пальца)
     - mobile: minSize = 44px (WCAG стандарт)
     - dense UI (таблицы, toolbars): minSize = 32px
     - primary actions: minSize = 48px (важные кнопки больше)

2. Определить тип элемента:
   if (element.tagName == 'button' && element.isPrimary) {
     minSize = 48px
   } else if (element.tagName == 'a' && element.isTextLink) {
     minSize = текущий lineHeight (ссылки в тексте)
   } else if (element.isIconButton) {
     minSize = 40px (иконки должны быть чуть меньше)
   }

3. Проверить touch target с учетом соседей:
   nearbyElements = getElementsWithin(element, 8px)
   if (nearbyElements.length > 0 && element.size < minSize + 8px) {
     Error: "Слишком близко к соседним элементам. Нужен отступ минимум 8px"
   }

4. Проверить соотношение сторон:
   aspectRatio = width / height
   if (isClickable && aspectRatio > 5 || aspectRatio < 0.2) {
     Warning: "Кликабельный элемент слишком вытянут (ratio: {aspectRatio})"
     Suggestion: "Оптимальное соотношение сторон: 1:1 до 3:1"
   }
```

### 2.2 Проверка overflow и clipping
**Алгоритм:**
```
1. Обнаружить overflow:
   if (content.width > container.width || content.height > container.height) {
     Issue: "Контент выходит за границы контейнера на {delta}px"
   }

2. Проверить overflow стили:
   if (overflow == 'hidden' && hasTextContent) {
     Warning: "Overflow hidden может скрыть текст. Используйте text-overflow: ellipsis"
   }

3. Обнаружить обрезанный текст:
   if (textLines > maxLines && overflow == 'hidden') {
     Error: "Текст обрезан без индикации (троеточия)"
     Suggestion: "Добавьте text-overflow: ellipsis или увеличьте высоту"
   }

4. Проверить scrollable контейнеры:
   if (overflow == 'auto' || overflow == 'scroll') {
     if (height < 150px) {
       Warning: "Слишком маленький scrollable контейнер (< 150px)"
     }
     if (!hasScrollbarStyling) {
       Info: "Добавьте кастомные стили для scrollbar для лучшего UX"
     }
   }
```

### 2.3 Золотое сечение и визуальная гармония
**Алгоритм:**
```
const PHI = 1.618 // Золотое сечение
const HARMONIOUS_RATIOS = [
  { ratio: 1.0, name: 'Квадрат' },
  { ratio: 1.414, name: 'A4 (√2)' },
  { ratio: 1.5, name: 'Классика (3:2)' },
  { ratio: PHI, name: 'Золотое сечение' },
  { ratio: 1.777, name: 'Широкоэкранный (16:9)' },
  { ratio: 2.0, name: 'Двойной квадрат' }
]

1. Вычислить соотношение сторон:
   elementRatio = width / height

2. Найти ближайшее гармоничное соотношение:
   closest = findClosest(elementRatio, HARMONIOUS_RATIOS)
   deviation = abs(elementRatio - closest.ratio)

3. Оценить гармоничность:
   if (deviation < 0.05) {
     // Хорошо! Близко к гармоничному соотношению
   } else if (deviation < 0.2) {
     Info: "Размеры близки к {closest.name} ({closest.ratio}). Отклонение: {deviation}"
   } else {
     Warning: "Негармоничное соотношение сторон: {elementRatio}. Рассмотрите {closest.name}"
   }
```

## 3. TYPOGRAPHY ANALYSIS 2.0 (Умная типографика)

### 3.1 Оптимальная длина строки (Line Length)
**Алгоритм:**
```
1. Подсчитать символы в строке:
   // Используем монопространный шрифт для расчета
   avgCharWidth = fontSize * 0.6 // приблизительно
   charsPerLine = containerWidth / avgCharWidth

2. Оценить читабельность:
   OPTIMAL_RANGE = {
     narrow: [45, 55],    // для узких колонок
     comfortable: [55, 75], // оптимум для большинства текстов
     wide: [75, 90]       // максимум для широких макетов
   }

   if (charsPerLine < 45) {
     Error: "Слишком короткие строки ({charsPerLine} символов). Минимум: 45"
     Suggestion: "Увеличьте ширину контейнера или уменьшите размер шрифта"
   } else if (charsPerLine > 90) {
     Error: "Слишком длинные строки ({charsPerLine} символов). Максимум: 90"
     Suggestion: "Уменьшите max-width до 75ch или разбейте на колонки"
   } else if (charsPerLine > 75) {
     Warning: "Длинные строки снижают читабельность. Рекомендуем 55-75 символов"
   }

3. Проверить line-height для длинных строк:
   if (charsPerLine > 70 && lineHeight < 1.6) {
     Warning: "Для длинных строк увеличьте line-height до 1.6-1.8"
   }
```

### 3.2 Иерархия шрифтов (Type Scale)
**Алгоритм:**
```
1. Собрать все размеры шрифтов на странице:
   fontSizes = [...new Set(getAllFontSizes())]

2. Проверить соответствие модульной шкале:
   // Идеальная шкала: каждый следующий размер = предыдущий * ratio
   MODULAR_SCALES = {
     'minor-second': 1.067,    // 16, 17, 18, 19...
     'major-second': 1.125,    // 16, 18, 20, 23... (Tailwind)
     'minor-third': 1.2,       // 16, 19, 23, 28...
     'major-third': 1.25,      // 16, 20, 25, 31...
     'perfect-fourth': 1.333,  // 16, 21, 28, 37...
     'golden-ratio': 1.618     // 16, 26, 42, 68...
   }

   bestScale = detectScale(fontSizes, MODULAR_SCALES)

3. Проверить отклонения:
   for each fontSize in fontSizes:
     expectedSize = baseSize * pow(bestScale.ratio, level)
     deviation = abs(fontSize - expectedSize)

     if (deviation > 2px) {
       Warning: "Размер {fontSize}px нарушает модульную шкалу {bestScale.name}"
       Suggestion: "Используйте {round(expectedSize)}px"
     }

4. Проверить минимальную разницу между уровнями:
   for (i = 0; i < fontSizes.length - 1; i++) {
     diff = fontSizes[i+1] - fontSizes[i]
     if (diff < 2px) {
       Warning: "Слишком маленькая разница между размерами ({diff}px)"
       Suggestion: "Минимальная разница должна быть 2-4px"
     }
   }
```

### 3.3 Line-height гармония
**Алгоритм:**
```
1. Определить оптимальный line-height на основе размера шрифта:
   optimalLineHeight(fontSize) {
     if (fontSize <= 12) return 1.5      // Мелкий текст
     if (fontSize <= 16) return 1.5      // Основной текст
     if (fontSize <= 24) return 1.4      // Подзаголовки
     if (fontSize <= 36) return 1.3      // Заголовки H3
     if (fontSize <= 48) return 1.2      // Заголовки H2
     return 1.1                          // Крупные заголовки H1
   }

2. Проверить соответствие:
   if (actualLineHeight < optimalLineHeight - 0.1) {
     Error: "Line-height слишком тесный ({actualLineHeight}). Рекомендуется {optimalLineHeight}"
   } else if (actualLineHeight > optimalLineHeight + 0.3) {
     Warning: "Line-height слишком разреженный ({actualLineHeight})"
   }

3. Проверить вертикальный ритм:
   // Line-height должен быть кратен базовой линии
   baselineGrid = 4px // или 8px
   lineHeightPx = fontSize * lineHeight

   if (lineHeightPx % baselineGrid != 0) {
     Info: "Line-height не кратен baseline grid ({baselineGrid}px)"
     Suggestion: "Используйте {round(lineHeightPx / baselineGrid) * baselineGrid}px"
   }
```

### 3.4 Проверка сиротских и висячих строк (Orphans & Widows)
**Алгоритм:**
```
1. Анализ параграфов:
   for each paragraph:
     lines = countLines(paragraph)

     // Проверка orphans (последняя строка на новой странице/колонке)
     if (orphans < 2) {
       Warning: "Orphans установлен в {orphans}. Минимум: 2 строки"
     }

     // Проверка widows (одно слово на последней строке)
     lastLine = getLastLine(paragraph)
     if (lastLine.words <= 1) {
       Info: "Висячее слово на последней строке"
       Suggestion: "Используйте &nbsp; или измените ширину контейнера"
     }

2. Проверка коротких последних строк:
   lastLineRatio = lastLine.width / paragraph.width
   if (lastLineRatio < 0.3) {
     Info: "Последняя строка слишком короткая ({lastLineRatio * 100}% ширины)"
   }
```

## 4. COLOR ANALYSIS 2.0 (Умная работа с цветом)

### 4.1 WCAG 3.0 APCA контрастность (Advanced Perceptual Contrast Algorithm)
**Проблема:** Старый WCAG 2.1 не учитывает восприятие цвета человеческим глазом.

**Алгоритм APCA:**
```
1. Вычислить APCA контраст:
   // Новая формула, учитывающая восприятие
   function calculateAPCA(textColor, bgColor) {
     // Конвертировать в линейное RGB
     textLum = getLuminance(textColor)
     bgLum = getLuminance(bgColor)

     // APCA формула
     if (bgLum > textLum) {
       // Светлый фон, темный текст
       contrast = (bgLum^0.56 - textLum^0.57) * 1.14
     } else {
       // Темный фон, светлый текст
       contrast = (bgLum^0.65 - textLum^0.62) * 1.14
     }

     return abs(contrast) * 100
   }

2. Проверить пороги APCA:
   APCA_THRESHOLDS = {
     body_text: {
       min: 60,      // Основной текст
       preferred: 75
     },
     heading_text: {
       min: 45,      // Заголовки (крупнее)
       preferred: 60
     },
     large_text: {
       min: 45,      // Текст > 24px
       preferred: 60
     },
     ui_components: {
       min: 15,      // Границы, иконки
       preferred: 30
     }
   }

3. Применить контекстные правила:
   if (fontSize < 14px) {
     minContrast = APCA_THRESHOLDS.body_text.preferred
   } else if (fontSize >= 24px) {
     minContrast = APCA_THRESHOLDS.large_text.min
   }

   if (apcaScore < minContrast) {
     Error: "APCA контраст {apcaScore} ниже минимума {minContrast}"
     Suggestion: getSuggestedColor(textColor, bgColor, minContrast)
   }
```

### 4.2 Цветовая гармония (Color Harmony)
**Алгоритм:**
```
1. Собрать цветовую палитру страницы:
   colors = extractColors(document)

2. Конвертировать в HSL для анализа:
   hslColors = colors.map(color => rgbToHsl(color))

3. Определить цветовую схему:
   scheme = detectColorScheme(hslColors):
     - Монохроматическая: все цвета в пределах 15° по hue
     - Аналоговая: цвета в пределах 30° по hue
     - Комплементарная: два цвета на противоположных сторонах (180° ±15°)
     - Триадная: три цвета на расстоянии 120° ±15°
     - Раздельно-комплементарная: три цвета, один основной + два соседних с комплементом
     - Тетрадная: четыре цвета, две комплементарных пары

4. Проверить соответствие схеме:
   if (scheme == 'unknown') {
     Warning: "Цвета не следуют известной цветовой схеме"
     Suggestion: "Рассмотрите использование {suggestScheme(colors)}"
   }

5. Проверить насыщенность:
   saturations = hslColors.map(c => c.saturation)
   if (stdDev(saturations) > 30) {
     Warning: "Слишком большой разброс насыщенности цветов"
     Suggestion: "Используйте близкие значения saturation для гармонии"
   }
```

### 4.3 Семантика цвета (Color Semantics)
**Алгоритм:**
```
1. Проверить семантическую корректность:
   COLOR_SEMANTICS = {
     error: ['red', 'rose'],       // Красные оттенки
     success: ['green', 'emerald'], // Зеленые
     warning: ['amber', 'orange'],  // Желто-оранжевые
     info: ['blue', 'sky'],         // Синие
     primary: ['blue', 'indigo'],   // Брендовые
   }

2. Обнаружить некорректное использование:
   if (element.role == 'error' && color NOT IN COLOR_SEMANTICS.error) {
     Error: "Элемент ошибки использует некорректный цвет ({color})"
     Suggestion: "Используйте красные оттенки для ошибок"
   }

3. Проверить дальтонизм (Color Blindness):
   // Симуляция протанопии (красно-зеленая слепота)
   simulatedColors = simulateColorBlindness(colors, 'protanopia')

   if (areColorsSimilar(simulatedColors)) {
     Error: "Цвета неразличимы для людей с протанопией"
     Suggestion: "Добавьте иконки или паттерны для дополнительной индикации"
   }

4. Проверить достаточность цветовых ролей:
   requiredRoles = ['primary', 'secondary', 'success', 'error', 'warning', 'info']
   missingRoles = requiredRoles.filter(role => !hasColorForRole(role))

   if (missingRoles.length > 0) {
     Warning: "Отсутствуют цвета для ролей: {missingRoles.join(', ')}"
   }
```

## 5. LAYOUT ANALYSIS (Новый модуль)

### 5.1 Проверка выравнивания (Alignment Detection)
**Алгоритм:**
```
1. Собрать все элементы с их границами:
   elements = getAllVisibleElements()
   edges = {
     left: elements.map(e => e.left),
     right: elements.map(e => e.right),
     top: elements.map(e => e.top),
     bottom: elements.map(e => e.bottom),
     centerX: elements.map(e => e.left + e.width/2),
     centerY: elements.map(e => e.top + e.height/2)
   }

2. Найти выравнивающие линии (alignment lines):
   TOLERANCE = 2px // Погрешность выравнивания

   for each edgeType in edges:
     clustered = clusterWithinTolerance(edges[edgeType], TOLERANCE)

     // Если 3+ элемента выравнены по одной линии
     alignmentLines = clustered.filter(cluster => cluster.length >= 3)

3. Обнаружить элементы, почти выровненные:
   for each element:
     nearbyLine = findNearestAlignmentLine(element, alignmentLines)
     distance = abs(element.edge - nearbyLine)

     if (distance > TOLERANCE && distance < 10px) {
       Warning: "Элемент почти выровнен по линии {nearbyLine}, но смещен на {distance}px"
       Suggestion: "Выровняйте элемент или намеренно увеличьте отступ"
     }

4. Проверить сетку:
   gridSize = detectGrid(elements) // Находит повторяющийся паттерн

   if (gridSize > 0) {
     for each element:
       if (element.left % gridSize != 0) {
         Info: "Элемент не выровнен по сетке {gridSize}px"
       }
   }
```

### 5.2 Z-index конфликты
**Алгоритм:**
```
1. Собрать все z-index значения:
   zIndexes = getAllZIndexes()
   sortedElements = sortByZIndex(elements)

2. Обнаружить перекрытия:
   for (i = 0; i < sortedElements.length - 1; i++) {
     current = sortedElements[i]
     next = sortedElements[i+1]

     if (isOverlapping(current.bounds, next.bounds)) {
       if (current.zIndex >= next.zIndex) {
         Warning: "Элемент с меньшим z-index ({current.zIndex}) перекрывает элемент с большим ({next.zIndex})"
       }
     }
   }

3. Проверить z-index шкалу:
   // Рекомендуемая шкала: 10, 20, 30... или 100, 200, 300...
   if (max(zIndexes) > 100) {
     // Проверить на использование 100, 200, 300...
     scale = 100

     scale = 10

     if (zIndex % scale != 0 && zIndex != 1) {
```

### 5.3 Визуальная иерархия (Visual Hierarchy)
**Алгоритм:**
```
1. Вычислить визуальный вес каждого элемента:
   visualWeight(element) {
     weight = 0

     // Размер
     weight += (element.width * element.height) / 1000

     // Яркость цвета (светлые цвета легче)
     weight += (1 - getLuminance(element.bgColor)) * 50

     // Насыщенность (яркие цвета тяжелее)
     weight += getSaturation(element.bgColor) * 30

     // Толщина границы
     weight += element.borderWidth * 5

     // Тень (элементы с тенью визуально тяжелее)
     if (element.hasShadow) weight += 20

     // Font weight
     weight += (element.fontWeight - 400) / 100 * 10

     // Позиция (верх тяжелее низа, лево тяжелее право)
     weight += (1 - element.top / viewportHeight) * 10

     return weight
   }

2. Проверить соответствие важности:
   semanticImportance = {
     h1: 100,
     h2: 80,
     h3: 60,
     button: 70,
     link: 40,
     p: 30
   }

   for each element:
     expectedWeight = semanticImportance[element.role]
     actualWeight = visualWeight(element)

     deviation = abs(expectedWeight - actualWeight)

     if (deviation > 30) {
       Warning: "Визуальный вес ({actualWeight}) не соответствует важности элемента ({element.role})"

       if (actualWeight > expectedWeight) {
         Suggestion: "Элемент слишком заметен для своей роли. Уменьшите размер/яркость"
       } else {
         Suggestion: "Элемент недостаточно заметен. Увеличьте размер/контраст"
       }
     }

3. Проверить фокальные точки:
   // Должна быть одна основная фокальная точка на экране
   focalPoints = elements.filter(e => visualWeight(e) > FOCAL_THRESHOLD)

   if (focalPoints.length == 0) {
     Warning: "Нет четкой фокальной точки на странице"
   } else if (focalPoints.length > 3) {
     Warning: "Слишком много фокальных точек ({focalPoints.length}). Пользователь не знает, куда смотреть"
   }
```

```javascript
 * Получить точные размеры текста в элементе

function getTextMetrics(element) {
  // Метод 1: Range API (самый точный)
  const range = document.createRange();
  range.selectNodeContents(element);
  const rect = range.getBoundingClientRect();

  return {
    width: rect.width,
    height: rect.height,
    // Точные границы текста
    top: rect.top,
    left: rect.left,
    right: rect.right,
    bottom: rect.bottom
  };
}

/**
 * Альтернатива: Canvas measureText (для одной строки)
 */
function measureTextWidth(text, font) {
  const canvas = document.createElement('canvas');
  const context = canvas.getContext('2d');
  context.font = font; // например: "16px Arial"

  const metrics = context.measureText(text);

  return {
    width: metrics.width,
    // Доступно в современных браузерах:
    actualBoundingBoxLeft: metrics.actualBoundingBoxLeft,
    actualBoundingBoxRight: metrics.actualBoundingBoxRight,
    actualBoundingBoxAscent: metrics.actualBoundingBoxAscent,
    actualBoundingBoxDescent: metrics.actualBoundingBoxDescent,
    // Полная ширина включая descenders (хвосты букв)
    totalWidth: metrics.actualBoundingBoxLeft + metrics.actualBoundingBoxRight
  };
}
```

```javascript
/**
 * Точный подсчет символов в каждой строке текста
 */
function getCharsPerLine(element) {
  const text = element.textContent;
  const computedStyle = window.getComputedStyle(element);
  const containerWidth = element.clientWidth;

  // Создаем временный элемент для измерения
  const testElement = document.createElement('span');
  testElement.style.cssText = `
    font-family: ${computedStyle.fontFamily};
    font-size: ${computedStyle.fontSize};
    font-weight: ${computedStyle.fontWeight};
    letter-spacing: ${computedStyle.letterSpacing};
    white-space: nowrap;
    visibility: hidden;
    position: absolute;
  `;
  document.body.appendChild(testElement);

  const lines = [];
  let currentLine = '';
  let currentWidth = 0;

  // Посимвольный подсчет
  for (let char of text) {
    testElement.textContent = currentLine + char;
    const newWidth = testElement.getBoundingClientRect().width;

    if (newWidth > containerWidth && currentLine.length > 0) {
      // Новая строка
      lines.push({
        text: currentLine,
        chars: currentLine.length,
        width: currentWidth
      });
      currentLine = char;
      currentWidth = newWidth - currentWidth;
    } else {
      currentLine += char;
      currentWidth = newWidth;
    }
  }

  // Последняя строка
  if (currentLine) {
    lines.push({
      text: currentLine,
      chars: currentLine.length,
      width: currentWidth
    });
  }

  document.body.removeChild(testElement);

  return {
    lines,
    totalLines: lines.length,
    avgCharsPerLine: lines.reduce((sum, l) => sum + l.chars, 0) / lines.length,
    maxCharsPerLine: Math.max(...lines.map(l => l.chars)),
    minCharsPerLine: Math.min(...lines.map(l => l.chars))
  };
}

/**
 * Упрощенный метод (приблизительный)
 */
function estimateCharsPerLine(element) {
  const style = window.getComputedStyle(element);
  const fontSize = parseFloat(style.fontSize);
  const containerWidth = element.clientWidth;

  // Средняя ширина символа = fontSize * 0.6 (для пропорциональных шрифтов)
  // Для моноширинных шрифтов = fontSize * 0.6
  const avgCharWidth = fontSize * 0.6;

  return Math.floor(containerWidth / avgCharWidth);
}
```

```javascript
/**
 * Точный подсчет строк текста
 */
function getLineCount(element) {
  const style = window.getComputedStyle(element);
  const lineHeight = parseFloat(style.lineHeight) || parseFloat(style.fontSize) * 1.2;
  const elementHeight = element.clientHeight;

  // Метод 1: По высоте
  const linesByHeight = Math.round(elementHeight / lineHeight);

  // Метод 2: Через Range API (более точный)
  const range = document.createRange();
  const textNode = element.firstChild;

  if (!textNode || textNode.nodeType !== Node.TEXT_NODE) {
    return linesByHeight;
  }

  let lines = 0;
  let previousTop = null;

  for (let i = 0; i < textNode.textContent.length; i++) {
    range.setStart(textNode, i);
    range.setEnd(textNode, i + 1);

    const rect = range.getBoundingClientRect();

    if (previousTop === null || rect.top > previousTop + 1) {
      lines++;
      previousTop = rect.top;
    }
  }

  return lines;
}

/**
 * Определить обрезанный текст (overflow hidden)
 */
function isTextTruncated(element) {
  // Метод 1: Сравнение scrollHeight с clientHeight
  if (element.scrollHeight > element.clientHeight) {
    return true;
  }

  // Метод 2: Проверка text-overflow: ellipsis
  const style = window.getComputedStyle(element);
```

```javascript
/**
 * Вычислить визуальный (оптический) центр элемента
 * Учитывает распределение визуального веса
 */
function getVisualCenter(element) {
  const rect = element.getBoundingClientRect();
  const style = window.getComputedStyle(element);

  // Геометрический центр
  const geometricCenter = {
    x: rect.left + rect.width / 2,
    y: rect.top + rect.height / 2
  };

  // Для текста: учитываем baseline и descenders
  if (element.textContent && element.textContent.trim()) {
    const fontSize = parseFloat(style.fontSize);
    const lineHeight = parseFloat(style.lineHeight) || fontSize * 1.2;

    // Descenders (хвосты букп g, j, p, q, y) занимают ~20% от высоты шрифта
    const descenderOffset = fontSize * 0.1;

    return {
      x: geometricCenter.x,
      y: geometricCenter.y - descenderOffset, // Сдвиг вверх
      isOpticallyAdjusted: true,
      adjustment: { x: 0, y: -descenderOffset }
    };
  }

  // Для изображений: анализ визуального веса
  if (element.tagName === 'IMG') {
    // Создаем canvas для анализа
    const canvas = document.createElement('canvas');
    const ctx = canvas.getContext('2d');
    canvas.width = element.naturalWidth;
    canvas.height = element.naturalHeight;

    ctx.drawImage(element, 0, 0);
    const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);

    // Вычисляем центр масс по яркости
    let totalWeight = 0;
    let weightedX = 0;
    let weightedY = 0;

    for (let y = 0; y < canvas.height; y++) {
      for (let x = 0; x < canvas.width; x++) {
        const i = (y * canvas.width + x) * 4;
        const brightness = (imageData.data[i] + imageData.data[i+1] + imageData.data[i+2]) / 3;

        // Темные пиксели тяжелее светлых
        const weight = 255 - brightness;

        totalWeight += weight;
        weightedX += x * weight;
        weightedY += y * weight;
      }
    }

    const opticalCenterX = rect.left + (weightedX / totalWeight) * (rect.width / canvas.width);
    const opticalCenterY = rect.top + (weightedY / totalWeight) * (rect.height / canvas.height);

    return {
      x: opticalCenterX,
      y: opticalCenterY,
      isOpticallyAdjusted: true,
      adjustment: {
        x: opticalCenterX - geometricCenter.x,
        y: opticalCenterY - geometricCenter.y
      }
    };
  }

  return {
    ...geometricCenter,
    isOpticallyAdjusted: false,
    adjustment: { x: 0, y: 0 }
  };
}
```

```javascript
/**
 * Нормализация цвета в единый формат
 */
function normalizeColor(color) {
  if (!color || color === 'transparent') {
    return { r: 0, g: 0, b: 0, a: 0, hex: 'transparent' };
  }

  // RGB / RGBA
  const rgbMatch = color.match(/rgba?\((\d+),\s*(\d+),\s*(\d+)(?:,\s*([\d.]+))?\)/);
  if (rgbMatch) {
    return {
      r: parseInt(rgbMatch[1]),
      g: parseInt(rgbMatch[2]),
      b: parseInt(rgbMatch[3]),
      a: rgbMatch[4] ? parseFloat(rgbMatch[4]) : 1,
      hex: rgbToHex(parseInt(rgbMatch[1]), parseInt(rgbMatch[2]), parseInt(rgbMatch[3]))
    };
  }

  // HEX
  const hexMatch = color.match(/^#([0-9a-f]{3}|[0-9a-f]{6})$/i);
  if (hexMatch) {
    const hex = hexMatch[1];
    const fullHex = hex.length === 3
      ? hex.split('').map(c => c + c).join('')
      : hex;

    return {
      r: parseInt(fullHex.substr(0, 2), 16),
      g: parseInt(fullHex.substr(2, 2), 16),
      b: parseInt(fullHex.substr(4, 2), 16),
      a: 1,
      hex: '#' + fullHex
    };
  }

  // Named colors
  const namedColors = {
    'white': { r: 255, g: 255, b: 255 },
    'black': { r: 0, g: 0, b: 0 },
    'red': { r: 255, g: 0, b: 0 },
    // ... добавить все CSS named colors
  };

  if (namedColors[color.toLowerCase()]) {
    const rgb = namedColors[color.toLowerCase()];
    return { ...rgb, a: 1, hex: rgbToHex(rgb.r, rgb.g, rgb.b) };
  }

  return null;
}

/**
 * Вычислить относительную яркость (для WCAG)
 */
function getRelativeLuminance(color) {
  // Конвертировать в линейное RGB
  const rsRGB = color.r / 255;
  const gsRGB = color.g / 255;
  const bsRGB = color.b / 255;

  const r = rsRGB <= 0.03928 ? rsRGB / 12.92 : Math.pow((rsRGB + 0.055) / 1.055, 2.4);
  const g = gsRGB <= 0.03928 ? gsRGB / 12.92 : Math.pow((gsRGB + 0.055) / 1.055, 2.4);
  const b = bsRGB <= 0.03928 ? bsRGB / 12.92 : Math.pow((bsRGB + 0.055) / 1.055, 2.4);

  return 0.2126 * r + 0.7152 * g + 0.0722 * b;
}

/**
 * WCAG 2.1 контрастность
 */
function calculateWCAGContrast(color1, color2) {
  const lum1 = getRelativeLuminance(color1);
  const lum2 = getRelativeLuminance(color2);

  const lighter = Math.max(lum1, lum2);
  const darker = Math.min(lum1, lum2);

  return (lighter + 0.05) / (darker + 0.05);
}

/**
 * WCAG 3.0 APCA контрастность (новый алгоритм)
 */
function calculateAPCAContrast(textColor, bgColor) {
  const txtY = getRelativeLuminance(textColor);
  const bgY = getRelativeLuminance(bgColor);

  // APCA параметры
  const normBG = 0.56;
  const normTXT = 0.57;
  const revTXT = 0.62;
  const revBG = 0.65;

  // Определить полярность (светлый на темном или наоборот)
  let Sapc;

  if (bgY > txtY) {
    // Светлый фон
    Sapc = (Math.pow(bgY, normBG) - Math.pow(txtY, normTXT)) * 1.14;
  } else {
    // Темный фон
    Sapc = (Math.pow(bgY, revBG) - Math.pow(txtY, revTXT)) * 1.14;
  }

  return Math.abs(Sapc) * 100;
}
```

```javascript
/**
 * RGB в HSL для анализа цветовой гармонии
 */
function rgbToHsl(r, g, b) {
  r /= 255;
  g /= 255;
  b /= 255;

  const max = Math.max(r, g, b);
  const min = Math.min(r, g, b);
  const diff = max - min;

  let h, s, l = (max + min) / 2;

  if (diff === 0) {
    h = s = 0; // achromatic
  } else {
    s = l > 0.5 ? diff / (2 - max - min) : diff / (max + min);

    switch (max) {
      case r: h = ((g - b) / diff + (g < b ? 6 : 0)) / 6; break;
      case g: h = ((b - r) / diff + 2) / 6; break;
      case b: h = ((r - g) / diff + 4) / 6; break;
    }
  }

  return {
    h: h * 360,
    s: s * 100,
    l: l * 100
  };
}

/**
 * Определить цветовую схему
 */
function detectColorScheme(colors) {
  const hues = colors.map(c => rgbToHsl(c.r, c.g, c.b).h);

  // Монохроматическая: все в пределах 15°
  if (Math.max(...hues) - Math.min(...hues) <= 15) {
    return { type: 'monochromatic', confidence: 0.9 };
  }

  // Аналоговая: в пределах 30°
  if (Math.max(...hues) - Math.min(...hues) <= 30) {
    return { type: 'analogous', confidence: 0.85 };
  }

  // Комплементарная: два цвета на противоположных сторонах
  if (hues.length === 2) {
    const diff = Math.abs(hues[0] - hues[1]);
    if (Math.abs(diff - 180) <= 15 || Math.abs(diff - 180 + 360) <= 15) {
      return { type: 'complementary', confidence: 0.9 };
    }
  }

  // Триадная: три цвета на расстоянии 120°
  if (hues.length === 3) {
    const sorted = hues.sort((a, b) => a - b);
    const diff1 = sorted[1] - sorted[0];
    const diff2 = sorted[2] - sorted[1];
    const diff3 = (360 + sorted[0]) - sorted[2];

    if (Math.abs(diff1 - 120) <= 15 && Math.abs(diff2 - 120) <= 15 && Math.abs(diff3 - 120) <= 15) {
      return { type: 'triadic', confidence: 0.9 };
    }
  }

  return { type: 'custom', confidence: 0.5 };
}
```

```javascript
function getUniqueSelector(element) {
```

```javascript
/**
 * Найти все элементы в радиусе

function getElementsInRadius(element, radius = 50) {
  const rect = element.getBoundingClientRect();
  const centerX = rect.left + rect.width / 2;
  const centerY = rect.top + rect.height / 2;

  const allElements = document.querySelectorAll('*');
  const nearby = [];

  for (let el of allElements) {
    if (el === element) continue;

    const elRect = el.getBoundingClientRect();
    const elCenterX = elRect.left + elRect.width / 2;
    const elCenterY = elRect.top + elRect.height / 2;

    const distance = Math.sqrt(
      Math.pow(elCenterX - centerX, 2) +
      Math.pow(elCenterY - centerY, 2)
    );

    if (distance <= radius) {
      nearby.push({
        element: el,
        distance,
        direction: getDirection(centerX, centerY, elCenterX, elCenterY)
      });
    }
  }

  return nearby.sort((a, b) => a.distance - b.distance);
}

/**
 * Определить направление от A к B
 */
function getDirection(x1, y1, x2, y2) {
  const angle = Math.atan2(y2 - y1, x2 - x1) * 180 / Math.PI;

  if (angle >= -45 && angle < 45) return 'right';
  if (angle >= 45 && angle < 135) return 'bottom';
  if (angle >= 135 || angle < -135) return 'left';
  return 'top';
}

/**
 * Проверить перекрытие элементов
 */
function checkOverlap(element1, element2) {
  const rect1 = element1.getBoundingClientRect();
  const rect2 = element2.getBoundingClientRect();

  const overlap = !(
    rect1.right < rect2.left ||
    rect1.left > rect2.right ||
    rect1.bottom < rect2.top ||
    rect1.top > rect2.bottom
  );

  if (!overlap) {
    return { overlaps: false, area: 0 };
  }

  // Вычислить площадь пересечения
  const overlapLeft = Math.max(rect1.left, rect2.left);
  const overlapRight = Math.min(rect1.right, rect2.right);
  const overlapTop = Math.max(rect1.top, rect2.top);
  const overlapBottom = Math.min(rect1.bottom, rect2.bottom);

  const overlapArea = (overlapRight - overlapLeft) * (overlapBottom - overlapTop);

  return {
    overlaps: true,
    area: overlapArea,
    percentage1: (overlapArea / (rect1.width * rect1.height)) * 100,
    percentage2: (overlapArea / (rect2.width * rect2.height)) * 100
  };
}
```

```javascript
      classes: Array.from(current.classList)
    });
    current = current.parentElement;
  }

  hierarchy.depth = depth;

  // Siblings (братья/сестры)
  if (element.parentElement) {
    hierarchy.siblings = Array.from(element.parentElement.children)
      .filter(el => el !== element)
      .map(el => ({
        element: el,
        tagName: el.tagName.toLowerCase(),
        id: el.id,
        classes: Array.from(el.classList)
      }));
  }

  // Descendants (потомки)
  function collectDescendants(el, level = 1) {
    const children = Array.from(el.children);

    for (let child of children) {
      hierarchy.descendants.push({
        element: child,
        level,
        tagName: child.tagName.toLowerCase(),
        id: child.id,
        classes: Array.from(child.classList)
      });

      collectDescendants(child, level + 1);
    }
  }

  collectDescendants(element);

  return hierarchy;
}
```

### 7. Viewport и responsive данные

#### 7.1 Получение viewport информации
```javascript
/**
 * Полная информация о viewport
 */
function getViewportInfo() {
  return {
    width: window.innerWidth,
    height: window.innerHeight,

    // Без scrollbars
    clientWidth: document.documentElement.clientWidth,
    clientHeight: document.documentElement.clientHeight,

    // С учетом всего документа
    scrollWidth: document.documentElement.scrollWidth,
    scrollHeight: document.documentElement.scrollHeight,

    // Текущий scroll
    scrollX: window.pageXOffset || document.documentElement.scrollLeft,
    scrollY: window.pageYOffset || document.documentElement.scrollTop,

    // Device pixel ratio (для Retina)
    devicePixelRatio: window.devicePixelRatio,

    // Orientation
    orientation: window.innerWidth > window.innerHeight ? 'landscape' : 'portrait',

    // Breakpoint detection
    breakpoint: detectBreakpoint(window.innerWidth),

    // Color scheme preference
    colorScheme: window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light',

    // Reduced motion
    reducedMotion: window.matchMedia('(prefers-reduced-motion: reduce)').matches,

    // Touch support
    touchSupport: 'ontouchstart' in window || navigator.maxTouchPoints > 0
  };
}

/**
 * Определить текущий breakpoint
 */
function detectBreakpoint(width) {
  if (width < 640) return 'xs';
  if (width < 768) return 'sm';
  if (width < 1024) return 'md';
  if (width < 1280) return 'lg';
  if (width < 1536) return 'xl';
  return '2xl';
}
```
